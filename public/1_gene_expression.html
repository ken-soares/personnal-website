Normalizing the data using CPM normalization¶

Normalization is a critical step to ensure comparability between samples. CPM (Counts Per Million) normalization scales raw gene counts by the library size (total reads per sample), thereby correcting for differences in sequencing depth and enabling meaningful comparison of gene expression levels across samples.

Filtering out genes with low expression in both conditions¶

At this stage, we remove genes that show very low expression levels in both conditions. These genes are unlikely to carry meaningful biological information, as their expression is close to background noise and may reflect technical variability rather than true biological signals.

Filtering them out helps to:

  • Reduce noise in the dataset
  • Improve the reliability of downstream analyses (e.g., differential expression)
  • Decrease the multiple testing burden, thereby increasing statistical power

This step is especially important in transcriptomics studies, such as idiopathic pulmonary fibrosis, where we aim to identify genes that are genuinely involved in disease-related processes rather than artifacts of low signal.

In a more advanced analysis pipeline, additional steps would typically be included, such as:

  • More refined filtering strategies
  • Data normalization methods
  • Batch effect correction
  • Integration of metadata (e.g., demographic or clinical variables)
  • Detection and careful evaluation of outliers

These steps help ensure that the observed patterns reflect true biological variation rather than technical or experimental biases.

Two-sample Kolmogorov–Smirnov test¶

The Kolmogorov–Smirnov (KS) test is a non-parametric test used to determine whether two samples are drawn from the same underlying distribution.

In our case, we use it to compare gene expression levels between control and IPF samples.

  • Null hypothesis (H₀): The control and IPF samples are drawn from the same continuous distribution.
  • Alternative hypothesis (H₁): The two samples are drawn from different distributions.

The KS statistic is defined as the maximum distance between the empirical cumulative distribution functions (ECDFs) of the two samples. A large KS statistic indicates a greater discrepancy between the distributions.

One advantage of this test is that it makes no assumption about the shape of the distribution (non-parametric), making it particularly suitable for gene expression data, which often deviate from normality.

A small p-value leads us to reject the null hypothesis, suggesting that gene expression distributions differ significantly between control and IPF samples.

It is important to note that the KS test is sensitive to any difference between distributions, including differences in mean, variance, or overall shape. For example, two normal distributions with the same mean but different variances will still be detected as significantly different.

While this sensitivity is a strength, it also means that the test does not indicate which aspect of the distributions differs. Therefore, additional analyses or visualizations are often needed to interpret the biological relevance of the observed differences.

For the purposes of this study, we use the KS test as a general tool to detect distributional differences between control and IPF gene expression profiles.

p_values = []
log2FCs = []
epsilon = 1 

for i in range(data.shape[0]):
    control = data[i, labels == "control"]
    ipf = data[i, labels == "ipf"]
    ks_stat, p_value = ks_2samp(control, ipf)
    
    p_values.append(p_value)
    # Since the KS test does not indicate which aspect of the distribution differs,
    # we complement it with a measure of central tendency difference between groups.
    # We compute the log2 fold change (log2FC), which compares the mean gene expression
    # between IPF and control samples.

    # First compute the fold change:
    # fold_change = mean(IPF) / mean(control)
    control_mean = np.mean(control)
    ipf_mean = np.mean(ipf)
    fold_change = (ipf_mean + epsilon) / (control_mean + epsilon) # the epsilon here is a small value used to avoid division by 0 errors
    # Then take the logarithm (base 2):
    # log2FC = log2(mean(IPF) / mean(control))
    log2FC = np.log2(fold_change)
    # The log2 transformation makes the measure symmetric around 0:
    # - log2FC > 0 → higher expression in IPF
    # - log2FC < 0 → lower expression in IPF
    # - log2FC = 0 → no difference
    log2FCs.append(log2FC)

# cast the results to numpy arrays
p_values = np.array(p_values)
log2FCs = np.array(log2FCs)
    
# This provides an interpretable measure of effect size for gene expression differences.

Avoiding false positives using Bonferroni Correction¶

When performing statistical tests across a large number of genes, the probability of obtaining false positives increases substantially. Even if each individual test is conducted with a significance level of α = 0.05, testing hundreds or thousands of genes means that some will appear significant purely by chance.

To address this issue, we apply the Bonferroni correction, a method that controls the family-wise error rate (FWER), i.e., the probability of making at least one false positive across all tests.

The Bonferroni correction adjusts p-values by multiplying them by the number of tests performed:

p_corrected = p_value × m

where m is the total number of genes tested.

# bonferroni correction to avoid false positives
p_values_bonf = p_values * len(genes)

Keeping samples of differentially expressed genes (DEGs)¶

To identify differentially expressed genes (DEGs), we apply both a statistical significance criterion and an effect size threshold.

First, we select genes that pass the Bonferroni-corrected significance threshold:

  • p_values_bonf <= 0.05 ensures that the observed difference in distribution (from the KS test) is statistically significant after correcting for multiple testing.

Second, we apply a threshold on the magnitude of expression change:

  • |log2FC| >= 2 ensures that the difference in expression is not only statistically significant but also biologically meaningful.
  • A log2 fold change of 2 corresponds to at least a 4-fold difference in expression between IPF and control samples.

These two criteria are combined using a logical AND.

Heatmap of Differentially Expressed Genes (DEGs)¶

This code generates a heatmap of DEGs using log2 fold changes relative to the mean expression in control samples.

Steps:¶

  1. Compute log2 fold change:
    For each significant gene, we calculate:

    $ \text{log2FC}_{i,j} = \log_2 (\frac{\text{expression}_{i,j} + \epsilon}{\text{mean control expression}_i + \epsilon}) $ where \$\epsilon = 1$ is a small constant to avoid division by zero.

  2. Plot heatmap with clustering:
    We use seaborn.clustermap to visualize gene expression patterns:

    • Red indicates higher expression than control mean
    • Blue indicates lower expression
    • Gray/neutral indicates similar expression

  3. Adjust labels:
    Font sizes for x and y labels are reduced for readability, especially for large matrices.

  4. Save figure:
    The heatmap is exported as a PDF (output/heatmap.pdf) for high-quality visualization.

Performance Note:¶

For large matrices, seaborn.clustermap uses hierarchical clustering via scipy by default, which can be slow. Installing the fastcluster library is recommended.

Scatter Plot of Gene Expression¶

This code generates a scatter plot showing the expression levels of a single gene (POSTN) across control and IPF samples.

Steps:¶

  1. Extract expression data:
    The expression values for POSTN are separated into control and IPF groups.

  2. Compute summary statistics:
    For each group:

    • Mean expression is calculated.
    • Standard error of the mean (SEM) is calculated as a measure of variability.

  3. Add jitter for visualization:
    A small random horizontal offset is applied to each point (add_jitter) to prevent overlap and make the distribution of points more visible.

  4. Plot individual points:

    • Control samples are shown in blue.
    • IPF samples are shown in red.
    • Point size is small (s=10) for clarity.

  5. Add error bars:

    • The black dots represent the mean expression for each group.
    • Error bars indicate the SEM, giving an idea of the variability around the mean.

  6. Customize axes and labels:

    • X-axis: groups (Control, IPF)
    • Y-axis: expression levels in CPM (counts per million)
    • Title reflects the gene being plotted.
    • Legend distinguishes the two groups.

Interpretation:¶

  • The scatter plot allows visualization of individual variability and the average expression between groups.
  • Red and blue points show the spread of expression, while error bars summarize central tendency and uncertainty.
  • For POSTN, we can visually assess whether expression differs between IPF and control samples.

Note: Using this type of scatter plot is particularly useful for genes of interest to inspect outliers, variance, and group-level differences beyond summary statistics like log2 fold change or p-values.

# expression scatter plot

gene_to_plot = "POSTN"

control_expression = data[genes == gene_to_plot, labels == "control"]
ipf_expression = data[genes == gene_to_plot, labels == "ipf"]

control_mean = np.mean(control_expression)
control_sem = np.std(control_expression) / np.sqrt(len(control_expression))
ipf_mean = np.mean(ipf_expression)
ipf_sem = np.std(ipf_expression) / np.sqrt(len(ipf_expression))

def add_jitter(values, jitter_amount=0.05):
    return values + np.random.uniform(-jitter_amount, jitter_amount, size=values.shape)

control_x = add_jitter(np.ones_like(control_expression) * 1)
ipf_x = add_jitter(np.ones_like(ipf_expression) * 2)

plt.scatter(control_x, control_expression, color='blue', label='Control', s=10)  # smaller dots with s=10
plt.scatter(ipf_x, ipf_expression, color='red', label='IPF', s=10)  # smaller dots with s=10

plt.errorbar(1, control_mean, yerr=control_sem, fmt='o', color='black', capsize=5)
plt.errorbar(2, ipf_mean, yerr=ipf_sem, fmt='o', color='black', capsize=5)

plt.xticks([1, 2], ['Control', 'IPF'])

plt.ylabel('Expression Level (CPM)')
plt.title(f'{gene_to_plot} Expression (CPM)')
plt.legend()
